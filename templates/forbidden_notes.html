1. A system uses a two-level paging scheme with a 4 KB page size. If the logical address space is 64-bit, and the first-level page table contains 512 entries, how many bits are used for the second-level page index?

Answer: 9 bits

Explanation: A 64-bit address space means we have 64 bits for addressing. A 4 KB page size means offset bits = log2(4096)=12 bits. The first-level page table has 512 entries, which requires log2(512)=9 bits. The remaining bits for the second-level index: 64−12−9=43 bits. Since this is a two-level paging scheme, the second-level page index should be 9 bits, matching the first-level index (assuming equal division).

2. A hard disk has 8 platters, each with 2 surfaces. If each surface has 1000 tracks, and each track contains 512 sectors, how many total sectors are available on the disk?

Answer: 8,192,000 sectors

Explanation: Platters: 8 Surfaces per platter: 2 → Total surfaces = 8×2=16 Tracks per surface: 1000 → Total tracks = 16×1000=16,000 Sectors per track: 512 Total sectors = 16,000×512=8,192,000

3. A system has a disk with an average seek time of 4 ms, a rotational speed of 10,000 RPM, and a transfer rate of 150 MB/s. How long does it take to read a 512-byte sector?

Answer: 0.1043 ms

Explanation: Average seek time = 4 ms Rotational latency: 60/10,000 seconds per revolution → Half rotation = 60/2×10,000=3 ms Transfer time for 512 bytes: 512/150×10^6 seconds = 0.00000341 ms Total access time ≈ Seek time + Rotational latency + Transfer time 4+3+0.00000341=7.00000341 ms (to locate and start transfer) Time to transfer a single sector is negligible compared to seek and latency.

4. A CPU has a pipeline with 5 stages. If each stage takes 1 ns and there are 100 instructions to execute, how long will it take assuming no stalls?

Answer: 104 ns

Explanation: The first instruction completes in 5 ns (5 pipeline stages). Each new instruction enters the pipeline every 1 ns after the first. Total time for 100 instructions: 5+(100−1)×1=104 ns.

5. A system has 16 GB of RAM and a page size of 2 MB. How many pages exist in the system?

Answer: 8192 pages

Explanation: 16 GB = 16×1024 MB = 16,384 MB Number of pages = 16,384/2=8192.

6. A CPU takes 300 cycles to perform a context switch. If the CPU operates at 2.5 GHz, how much time (in microseconds) is spent during one context switch?

Answer: 0.12 μs

Explanation: CPU clock period = 1/2.5×10^9 = 0.4 ns per cycle. Context switch takes 300 cycles → 300×0.4=120 ns = 0.12 μs.

7. A system has a TLB with 64 entries. If the page size is 8 KB and the system has 4 GB of RAM, what is the TLB coverage (total memory mapped by TLB)?

Answer: 512 MB

Explanation: Each TLB entry maps one page. Each page = 8 KB TLB coverage = 64×8 KB = 512 KB Since each entry maps 64 pages in the worst case, total memory mapped = 512 MB.

8. A virtual memory system uses a 3-level page table. The page size is 4 KB, and each table entry is 8 bytes. If a 64-bit system is used, how much memory is required to store a fully populated page table?

Answer: 256 TB

Explanation: 64-bit address → 4 KB pages → 12-bit offset. Remaining address bits: 64−12=52 bits. Divided into 3 levels: 52 bits → 3 tables → 52/3≈17 bits per table. Each table has 2^17 entries = 128K entries. Each entry = 8 bytes. Memory per table = 2^17×8=1 MB. Total memory = 3×2^17×8=256 TB (if fully populated).

9. A RAID-5 system has 5 disks, each with 2 TB of storage. What is the usable capacity?

Answer: 8 TB

Explanation: RAID-5 uses 1 disk for parity. Total storage = 5 × 2 TB = 10 TB Usable storage = 10−2=8 TB (since 1 disk is used for parity).

10. A system with 16 processors has a parallel program that executes in 10 seconds on a single core. If the program is perfectly parallelizable, what is the expected execution time on 16 cores?

Answer: 0.625 seconds

Explanation: Amdahl’s Law (assuming perfect scaling): Execution time = Serial Time / Number of Cores 10/16=0.625 seconds.
